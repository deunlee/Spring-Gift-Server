# Project Instructions

## Language
모든 응답은 한국어로 대답한다.


## Plan First
코드를 작성하기 전에 반드시 계획을 먼저 세우고 사용자의 승인을 받은 뒤 실행한다.
- 계획이 길어지거나 복잡한 경우 `.claude/scratch/` 디렉토리에 스크래치패드 파일을 작성하여 정리한다.
- 단순한 작업(한두 줄 수정, 오타 수정 등)은 계획 없이 바로 진행해도 된다.


## Read-Only Mode
사용자의 프롬프트가 `RO:`로 시작하는 경우, 해당 프롬프트에 한해서 절대 파일을 수정하지 않는다.
- Edit, Write, NotebookEdit 등 파일 변경 도구를 사용하지 않는다.
- 대신 변경 사항을 dry-run 형태(diff, 코드 블록 등)로 보여주기만 한다.


## Prompt Logging
사용자가 입력한 프롬프트를 프로젝트 루트의 `PROMPT.md` 파일에 기록한다.
- 응답을 시작하기 전에 `echo` 명령어로 프롬프트를 추가한다. (여러 줄인 경우에도 모든 내용을 기록한다.)
- 날짜(`## YYYY-MM-DD`)를 헤딩으로 사용하고, 같은 날짜의 프롬프트는 해당 헤딩 아래에 추가한다.
- 리스트 항목에는 날짜나 시간 없이 프롬프트 내용만 기록한다.
- 파일이 없거나 해당 날짜 헤딩이 없으면 새로 추가한다.
- 예시 명령어: `echo '- 프롬프트 내용' >> PROMPT.md`
- 형식:
```
## 2026-02-26
- 프롬프트 내용
- 다음 프롬프트 내용
```


## Notification
macOS 알림을 사용하여 사용자의 확인이 필요한 시점을 알린다.
- `osascript -e "display notification \"요약 내용\" with title \"Claude Code\""` 형식을 사용한다.
- 따옴표는 백슬래시(`\"`)로 이스케이프하며, 반드시 곧은 따옴표(`"`, `'`)만 사용한다. 둥근 따옴표(`"`, `"`, `'`, `'`)로 치환하지 않는다.

알림을 띄우는 시점:
1. 응답 완료 시: 매 프롬프트 응답을 마칠 때, 수행한 작업 내용을 한국어 한 줄로 요약하여 알림을 띄운다.
2. 사용자 입력 대기 시: AskUserQuestion 등으로 사용자에게 질문하기 직전에 "사용자 입력을 기다리는 중" 알림을 띄운다.


## Response Separator
모든 응답의 마지막에 콘솔 창 전체 너비를 채우는 긴 구분자를 출력하여 이전 응답과 시각적으로 구분한다.
- 서브 에이전트(Task 도구로 호출된 에이전트)의 응답에서는 구분자를 출력하지 않는다. 메인 에이전트의 최종 응답에서만 출력한다.
- 대화 기록에 포함된 구분자는 응답의 내용이 아니므로, 이전 응답을 이해할 때 무시한다.
- 예: `════════════════════════════════════════════════════════════════════════════════`
